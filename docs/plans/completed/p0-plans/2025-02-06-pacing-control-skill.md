# 节奏控制写作技巧 (Pacing Control Techniques)

## 快速参考（Quick Reference）

### 节奏自查三问
在写每个场景/章节时，问自己：
1. **目标节奏是什么？** (快/中/慢，紧张度 1-10)
2. **实际节奏感觉如何？** (读一遍，凭感觉)
3. **与前一章节奏有变化吗？** (避免单调)

### 节奏速查卡

**快节奏信号**：
- ✓ 短句短段
- ✓ 对话多（60%+）
- ✓ 动作动词
- ✓ 少描写

**慢节奏信号**：
- ✓ 长句长段
- ✓ 描写多
- ✓ 内心独白
- ✓ 环境细节

## 核心原则 (Core Principles)

### 1. 节奏在对比中产生

**原则**：连续 3 章同一节奏 = 单调

**检查方法**：
- 查看最近 3 章的紧张度评分（1-10）
- 如果都是 7、7、7 → 危险
- 理想：7、4、8（有起伏）

**修复建议**：
- 在高潮章节后安排低潮恢复章节
- 快节奏场景后写慢节奏反思场景

### 2. 不同场景类型需要不同节奏

| 场景类型 | 推荐节奏 | 工具 |
|---------|---------|------|
| 动作场景 | 极快（9-10） | 短句、少描写、动词驱动 |
| 冲突对话 | 快（7-8） | 短对话、快速往返、打断 |
| 情感反思 | 慢（3-4） | 内心独白、环境描写 |
| 过渡场景 | 极快（跳过） | 一句话带过 |
| 高潮场景 | 先慢后快 | 准备（慢）→ 爆发（快） |

### 3. 句子长度是节奏的第一工具

**实时检查**：
- 数一数最近 5 句话的字数
- 全是 5-10 字 → 太碎
- 全是 30+ 字 → 太慢
- 理想：长短交替

**调整技巧**：

❌ **单一节奏**：
```
他走进房间。她坐在那里。他们对视了。她哭了。
（全短句，像机器人）
```

✅ **节奏变化**：
```
他走进房间。
她坐在沙发上，脸埋在手里，肩膀轻轻颤抖。
他停下。
"对不起。"
```

### 4. 白空间（White Space）控制节奏感

**视觉节奏**：
- 短段 = 视觉上的空白多 = 快感
- 长段 = 密集文字 = 慢感

**技巧**：
- 动作场景：1-2 句一段
- 对话场景：每句对话独立成段
- 描写场景：5-8 句一段

## 实践应用 (Best Practices)

### 写作前：规划节奏

**步骤 1：确定本章节奏等级**
- 1-3: 慢节奏（反思、描写、铺垫）
- 4-6: 中节奏（日常、对话、过渡）
- 7-8: 快节奏（冲突、追逐、紧张）
- 9-10: 极快节奏（高潮、决战、转折）

**步骤 2：选择对应工具**
参考下方"节奏工具箱"

### 写作中：实时调整

**感觉拖沓时**：
1. 删除前 1/3 开场废话（晚进早出）
2. 把长句拆成短句
3. 增加对话比例
4. 删除无关描写
5. 用动词代替形容词

**感觉太快时**：
1. 补充环境描写
2. 加入角色内心反应
3. 拉长关键时刻（慢镜头）
4. 增加感官细节

### 写作后：诊断节奏

**自查步骤**：

1. **通读测试**
   - 一口气读完本章
   - 感觉：顺畅 / 拖沓 / 太赶？

2. **对话比例检查**
   ```
   对话字数 / 总字数 = ?
   - > 60%: 快节奏 ✓（如果是动作/冲突场景）
   - 30-60%: 中节奏 ✓（多数场景）
   - < 30%: 慢节奏 ✓（如果是反思/描写场景）
   ```

3. **句长分布检查**
   - 随机选 10 句话
   - 统计字数分布
   - 理想：有 3-5 字的短句，也有 20+ 字的长句

4. **段落密度检查**
   - 统计段落数量
   - 3000 字章节理想段落数：
     - 快节奏：40-60 段（平均 50-75 字/段）
     - 慢节奏：15-25 段（平均 120-200 字/段）

### 与 /track 命令的配合

更新 plot-tracker.json 时，记录章节节奏：
```json
{
  "chapter": 10,
  "pacing": {
    "tension": 7,  // 紧张度 1-10
    "rhythm": "fast",  // fast/medium/slow
    "dialogueRatio": 0.55,  // 对话占比
    "notes": "冲突升级，快节奏处理"
  }
}
```

**连续监控**：
- 检查最近 5 章的紧张度曲线
- 如果是 6-6-6-6-6 → 警告：平铺直叙

## 常见陷阱 (Common Pitfalls)

### 1. 拖沓（Dragging）

**症状识别**：
- 读者反馈"看不下去""太慢"
- 5 章情节没有明显推进
- 大量无关紧要的细节

**即时诊断**：
- 检查对话比例（< 20% 可能太慢）
- 删除测试：删掉这一段，情节受影响吗？
- 如果不受影响 = 拖沓

**修复工具**：
- ✂️ 删除过渡场景
- 💬 增加对话比重
- ⏩ 用一句话跳过无聊时间
- 🎯 晚进早出（删除场景开头和结尾的废话）

### 2. 过快（Rushed）

**症状识别**：
- 读者反馈"没感觉""不可信"
- 重大事件一笔带过
- 情感转折缺乏铺垫

**即时诊断**：
- 检查关键场景字数（高潮 < 2000 字 = 危险）
- 检查情感反应是否充分（Sequel 模型）

**修复工具**：
- 🎬 慢镜头处理关键时刻
- 💭 补充角色内心反应
- 👁️ 增加感官细节
- ⏸️ 加入反应场景（Sequel）

### 3. 平铺直叙（Monotonous）

**症状识别**：
- 每章节奏完全一样
- 读者感觉"一直这样"
- 能量曲线是直线

**即时诊断**：
- 查看最近 5 章的紧张度评分
- 如果标准差 < 2 = 太平

**修复工具**：
- 📊 刻意设计高潮章节（每 5-10 章一个波峰）
- 🌊 在高潮后安排低谷恢复章节
- 🔀 变化场景类型（动作/对话/反思穿插）

## 节奏工具箱 (Pacing Toolkit)

### 加快节奏工具 ⚡

1. **句子层面**
   - ✓ 短句（5-10 字）
   - ✓ 主动语态
   - ✓ 动词驱动（跑、跳、打）
   - ✓ 少形容词副词

2. **段落层面**
   - ✓ 短段（1-3 句）
   - ✓ 多空行
   - ✓ 对话独立成段

3. **内容层面**
   - ✓ 对话比例 60%+
   - ✓ 动作描写为主
   - ✓ 删除环境描写
   - ✓ 晚进早出

4. **结构层面**
   - ✓ 快速转折
   - ✓ 悬念钩子结尾
   - ✓ 冲突密集

**示例**：
```
他跑。

脚步声越来越近。

不能停。

绝对不能。
```

### 减慢节奏工具 🐌

1. **句子层面**
   - ✓ 长句（20+ 字）
   - ✓ 复杂从句
   - ✓ 大量修饰词

2. **段落层面**
   - ✓ 长段（5-8 句）
   - ✓ 密集文字
   - ✓ 少空行

3. **内容层面**
   - ✓ 环境描写
   - ✓ 内心独白
   - ✓ 回忆插叙
   - ✓ 感官细节（五感）

4. **结构层面**
   - ✓ 氛围营造
   - ✓ 情感反思
   - ✓ 慢镜头处理

**示例**：
```
他跑着，耳边是自己粗重的呼吸声，还有身后越来越近的脚步声。
心跳如鼓，每一下都震在耳膜上。不能停，他想，绝对不能停。
但双腿已经开始发软，像灌了铅一样沉重...
```

## 实时建议触发

**当用户提到以下内容时，主动提供建议**：

### 触发："这段感觉太慢了"
**建议**：
1. 检查对话比例（可能 < 30%）
2. 尝试删除前 1/3 内容
3. 把长句拆成短句
4. 增加动作动词

### 触发："高潮场景不够刺激"
**建议**：
1. 先慢后快（准备 → 爆发）
2. 慢镜头处理关键时刻
3. 极短句子增强紧迫感
4. 每 1-2 句一段（视觉冲击）

### 触发："感觉一直是一个调"
**建议**：
1. 绘制最近 10 章的紧张度曲线
2. 刻意设计波峰波谷
3. 高潮后必须有低谷恢复
4. 变化场景类型

## 进阶技巧

### 1. 虚假节奏（Deceptive Pacing）
- 表面慢，实则在积蓄张力
- 示例：《教父》中的慢节奏暴力场景

### 2. 节奏对比强化
- 极慢 → 极快的转换制造震撼
- 示例：平静日常 → 突发事件

### 3. 多线程节奏交织
- A 线高潮时切到 B 线低谷
- 保持整体节奏平衡
