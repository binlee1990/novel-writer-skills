### 扩展 1：读者期待识别

**从已写内容中识别读者可能形成的期待**：

#### 期待来源

| 来源 | 描述 | 示例 |
|------|------|------|
| 伏笔 | 作者埋下的线索 | 提到神秘宝物 → 读者期待主角获得 |
| 承诺 | 角色做出的承诺/誓言 | 「我一定会回来」→ 读者期待重逢 |
| 模式 | 已建立的叙事模式 | 每次遇到强敌都会突破 → 读者期待下次也突破 |
| 冲突 | 未解决的冲突 | 主角与反派结仇 → 读者期待复仇 |
| 类型惯例 | 该类型的常见套路 | 玄幻文 → 读者期待升级打脸 |

#### 检测方法

扫描已写章节，提取：
1. 所有未回收的伏笔 → 转化为「读者期待列表」
2. 所有角色承诺/誓言 → 转化为「承诺期待列表」
3. 已建立的叙事模式 → 转化为「模式期待列表」
4. 未解决的冲突 → 转化为「冲突期待列表」

#### 输出格式

```
📋 读者期待清单
━━━━━━━━━━━━━━━━━━━━

🔮 伏笔期待：
1. [伏笔内容] — 埋设于第 [N] 章，紧急度 [X]
2. [伏笔内容] — 埋设于第 [M] 章，紧急度 [Y]

🤝 承诺期待：
1. [角色A] 承诺 [内容] — 第 [N] 章，未兑现
2. [角色B] 承诺 [内容] — 第 [M] 章，未兑现

🔄 模式期待：
1. 读者期待 [模式描述]（已出现 [N] 次）
2. 读者期待 [模式描述]（已出现 [M] 次）

⚔️ 冲突期待：
1. [冲突描述] — 未解决，持续 [N] 章
2. [冲突描述] — 未解决，持续 [M] 章
```

### 扩展 2：满足/颠覆期待的节奏控制

**指导何时满足期待、何时颠覆期待**：

#### 满足/颠覆比例

**推荐比例**：70% 满足 + 30% 颠覆

| 策略 | 适用场景 | 效果 |
|------|---------|------|
| 直接满足 | 读者等待已久的期待 | 满足感、爽感 |
| 超越满足 | 重要期待 | 惊喜 + 满足（比期待的更好） |
| 延迟满足 | 期待紧急度中等 | 增加期待感 |
| 部分满足 | 复杂期待 | 满足一部分，保留悬念 |
| 颠覆 | 模式期待（已重复 3+ 次） | 意外感、新鲜感 |
| 假颠覆 | 重要期待 | 先打破再以更好的方式满足 |

#### 节奏建议

```
📊 期待管理建议
━━━━━━━━━━━━━━━━━━━━

当前待处理期待：[N] 个
紧急期待（紧急度 > 0.8）：[M] 个

建议操作：
1. 🟢 满足：[期待内容]（已等待 [N] 章，读者耐心接近极限）
2. 🟡 延迟：[期待内容]（紧急度中等，可再等 [N] 章）
3. 🔴 颠覆：[期待内容]（模式已重复 [N] 次，适合打破）

⚠️ 警告：
- [期待A] 已延迟 [N] 章，再不满足读者可能弃书
- [期待B] 已满足 3 次同类期待，下次考虑颠覆
```

### 扩展 3：读者情绪预测

**基于当前章节内容，预测读者的情绪反应**：

#### 情绪维度

| 维度 | 正面 | 负面 |
|------|------|------|
| 兴奋度 | 热血、激动、期待 | 无聊、疲倦 |
| 满足度 | 爽快、满意、感动 | 失望、不满 |
| 紧张度 | 紧张、担忧、悬念 | 松弛、无感 |
| 认同度 | 共鸣、支持、理解 | 反感、不认同 |

#### 输出格式

```
😊 读者情绪预测 — 第 [N] 章
━━━━━━━━━━━━━━━━━━━━

兴奋度：████████░░ 高（战斗场景密集）
满足度：██████░░░░ 中（有小爽点但无大满足）
紧张度：█████████░ 高（悬念未解）
认同度：███████░░░ 中高（主角行为合理）

整体情绪：积极偏高
风险点：兴奋度和紧张度同时高，下一章需要释放（否则读者疲劳）

建议：
- 下一章安排一个满足感高的场景（回收伏笔/打脸/突破）
- 适当降低紧张度（解决一个悬念）
- 保持认同度（主角行为继续合理）
```
