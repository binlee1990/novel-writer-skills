### Task 18.1: 在写作方法选择中添加网文结构

**Files:**
- Modify: `templates/commands/plan.md`（2.1 写作方法选择部分）

**Step 1: 在现有写作方法列表之后添加网文专用结构**

在 `plan.md` 的 `#### 2.1 写作方法选择` 中，在「类型专用结构」条目之后，展开为详细的网文结构模板：

```markdown

##### 网文专用结构模板

**适用条件**：`specification.md` 中的类型为网文类（玄幻、都市、言情、游戏文等）

###### 结构 A：升级流（最常用）

**核心循环**：修炼/积累 → 突破/升级 → 展示/打脸 → 新挑战

**卷结构模板**：
```
每卷 20-40 章，遵循以下节奏：

开篇（1-3 章）：
  - 进入新环境/新地图
  - 展示当前实力（让读者有基准）
  - 遭遇新挑战/新对手

发展（4-15 章）：
  - 修炼/积累（穿插小冲突）
  - 获得机缘/资源
  - 小规模打脸（1-2 次）
  - 结识新角色/势力

升级（16-20 章）：
  - 突破瓶颈
  - 境界提升
  - 能力展示

高潮（21-30 章）：
  - 大规模冲突
  - 碾压式打脸
  - 解决本卷核心矛盾

收束（31-35 章）：
  - 收获奖励
  - 为下一卷埋伏笔
  - 引出新地图/新挑战
```

**爽点分布**：每 3-5 章一个小爽点，每 10-15 章一个大爽点

###### 结构 B：副本流

**核心循环**：进入副本 → 探索/解谜 → Boss 战 → 奖励/升级

**卷结构模板**：
```
每卷 = 1 个副本（15-30 章）

准备期（1-3 章）：
  - 副本信息获取
  - 组队/准备
  - 进入副本

探索期（4-12 章）：
  - 副本环境描写
  - 小怪/陷阱/谜题
  - 队友互动/冲突
  - 获得线索/道具

Boss 战（13-20 章）：
  - Boss 登场（展示压倒性实力）
  - 苦战（队友受伤/计划失败）
  - 转机（主角发现弱点/觉醒新能力）
  - 击败 Boss

收获期（21-25 章）：
  - 副本奖励分配
  - 升级/进化
  - 副本外的影响（名声/敌人）
  - 下一个副本的线索
```

**爽点分布**：探索期每 2-3 章一个发现，Boss 战集中爽点

###### 结构 C：任务流

**核心循环**：接任务 → 执行 → 完成 → 奖励 → 新任务

**卷结构模板**：
```
每卷包含 1 个主线任务 + 2-3 个支线任务

主线任务（贯穿全卷）：
  - 任务发布（1-2 章）
  - 调查/准备（3-5 章）
  - 执行（6-15 章，穿插支线）
  - 意外/转折（16-20 章）
  - 完成/失败（21-25 章）

支线任务（穿插在主线中）：
  - 每个支线 3-5 章
  - 提供资源/信息/人脉
  - 与主线有隐藏关联
```

###### 结构 D：日常流

**核心循环**：日常事件 → 角色互动 → 小冲突 → 成长/变化

**卷结构模板**：
```
每卷围绕一个主题（如：学院生活、开店经营、种田发展）

日常展开（1-10 章）：
  - 日常生活描写
  - 角色互动（重点）
  - 轻松幽默的小事件
  - 缓慢但持续的成长

冲突引入（11-15 章）：
  - 打破日常的事件
  - 外部威胁或内部矛盾
  - 角色关系考验

解决与回归（16-20 章）：
  - 解决冲突
  - 关系深化
  - 回归日常（但有变化）
  - 新的日常元素引入
```

**爽点分布**：以情感满足和角色互动为主，战斗爽点为辅

##### 结构选择建议

| 故事类型 | 推荐结构 | 备选结构 |
|---------|---------|---------|
| 玄幻/修仙 | 升级流 | 副本流 |
| 都市异能 | 任务流 | 升级流 |
| 游戏/系统文 | 副本流 | 任务流 |
| 种田/经营 | 日常流 | 任务流 |
| 言情 | 日常流 | 任务流 |
| 混合型 | 主线升级流 + 支线副本流 | — |
```
