# 多线程叙事追踪 (Multi-Thread Narrative Tracking)

## 概述

多线程叙事追踪 Skill 帮助作者管理复杂小说中的多条平行故事线。它将"线程是否平衡"这个模糊问题转化为可量化、可追踪的分析框架。

**位置**: `templates/skills/writing-techniques/multi-thread-narrative/`

## 什么是多线程叙事？

复杂小说常有多条平行故事线：
- **多POV**：不同角色的视点章节（如《权力的游戏》）
- **多地点**：同时发生在不同地方的事件
- **多时间线**：现在、回忆、预言等平行时间
- **多主题线**：独立的副情节和主题

## 为什么要管理线程？

| 管理得当 | 管理不当 |
|---------|---------|
| 线程交替有序 | 进度失衡，某线被遗忘 |
| 切换制造悬念 | 切换混乱，读者困惑 |
| 汇聚产生高潮 | 伏笔遗忘，线索断裂 |

## 五大核心能力

| 能力 | 说明 |
|------|------|
| 线程识别 | 自动识别 POV/地点/时间线/主题线 |
| 进度追踪 | 章节数、字数占比、最后更新、弧线阶段 |
| 平衡分析 | 热线/冷线检测、失衡警告 |
| 交织建议 | 切换时机、悬念设计、汇聚点规划 |
| 伏笔追踪 | 线内/线间/全局伏笔的回收状态 |

## 线程状态

| 状态 | 条件 | 图标 |
|------|------|------|
| 活跃 | 冷线章数 ≤ 3 | ✅ |
| 注意 | 冷线章数 4-5 | ⚠️ |
| 冷线 | 冷线章数 6-8 | ❄️ |
| 危险 | 冷线章数 > 8 | 🔴 |
| 过热 | 连续 3+ 章出现 | 🔥 |

## 四种交织策略

### 1. 悬念点切换
在悬念点切换到另一线，制造翻页欲望。

### 2. 平行升级
多线同步升级张力，汇聚到高潮。

### 3. 呼应切换
一线埋设，另一线呼应，通过主题关联。

### 4. 时间同步切换
不同线程在同一时间点切换，制造紧迫感。

```
📊 线程总览

3. [布兰线] POV | 2章 (8%) | 🔴 危险(8章未更新)

⚖️ 平衡性: 6.5/10
🔴 布兰线已 8 章未更新

💡 下一章建议: 切换到【布兰线】
- 理由: 冷线回暖 + 琼恩线悬念点适合切换
- 衔接: 布兰在北方，可呼应琼恩的异鬼发现

🔍 伏笔追踪
- [线间] 神秘符号: 琼恩线→布兰线 | 🟠 High

```

## 使用方式

### 自动激活
当检测到故事有 2+ 条独立线程时自动激活：
- 多个 POV 角色
- 并行地点或时间线
- 独立的副情节

### 与其他命令配合

| 命令/Skill | 配合方式 |
|-----------|---------|
| `/plan` | 在大纲阶段设计线程分配和汇聚点 |
| `/analyze` | 提供线程维度的分析数据 |
| `/revise` | 结构修改时调整线程平衡 |
| `reader-expectation` | 每条线的期待独立追踪 |
| `pacing-control` | 各线独立节奏 + 整体切换节奏 |
| `character-arc` | 每条 POV 线对应一个角色弧线 |

## 最佳实践

1. **至少 3 章后启用**：需要足够数据识别线程
2. **叙事优先于平衡**：不为平衡牺牲节奏
3. **标记重要伏笔**：尤其是线间伏笔，最容易遗忘
4. **定期检查**：每 5-10 章运行一次线程分析
5. **控制线程数量**：同时活跃线程不超过 6 条

## 线程数量建议

| 篇幅 | 建议线程数 |
|------|-----------|
| 短篇 (< 5万字) | 1-2 条 |
| 中篇 (5-15万字) | 2-3 条 |
| 长篇 (15-50万字) | 3-5 条 |
| 超长篇 (> 50万字) | 4-6 条 |
