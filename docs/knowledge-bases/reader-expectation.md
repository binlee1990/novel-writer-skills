# 读者期待管理 (Reader Expectation Management)

## 概述

读者期待管理 Skill 帮助作者识别、追踪并策略性地满足读者期待。它将"读者会不会失望"这个抽象问题转化为可量化、可操作的分析框架。

**位置**: `templates/skills/writing-techniques/reader-expectation/`

## 什么是读者期待？

读者期待 = 读者预期故事中会发生的事情。

**来源**：
- **明确承诺**："总有一天我要打败他!" → 期待决战
- **类型惯例**：爽文读者期待主角变强、打脸
- **铺垫暗示**："这把剑似乎不简单" → 期待剑的秘密揭晓
- **角色目标**："我要找到父亲" → 期待寻亲结果

## 为什么要管理期待？

| 管理得当 | 管理不当 |
|---------|---------|
| 读者有满足感 | 期待落空 → 失望弃书 |
| 节奏紧凑有力 | 过早满足 → 失去悬念 |
| 高潮令人震撼 | 遗忘承诺 → 情节漏洞 |

## 五种期待类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 情节承诺 | 明确承诺会发生的事件 | "我发誓要打败他!" |
| 谜题悬念 | 需要解答的悬念 | "那个人究竟是谁?" |
| 角色成长 | 角色发展的期待 | 弱者主角 → 期待变强 |
| 关系弧线 | 人物关系的发展 | 敌对二人 → 期待和解 |
| 世界观谜题 | 世界观层面的疑问 | "这个世界的真相?" |

## 紧迫性计算

每个期待都有紧迫性分数，基于：

```
紧迫性 = (章节间隔 / 体裁阈值) × 铺垫强度 × 类型权重
```

| 分数 | 等级 | 建议行动 |
|------|------|---------|
| < 0.3 | 🟢 Low | 可继续延迟，偶尔提及 |
| 0.3-0.7 | 🟡 Medium | 需要进展或新线索 |
| 0.7-1.0 | 🟠 High | 3-5 章内必须有进展 |
| > 1.0 | 🔴 Critical | 立即安排满足 |

**体裁阈值**（读者耐心上限）：
- 爽文/升级流：10 章
- 都市/现实：15 章
- 悬疑/推理：20 章
- 史诗奇幻：30 章

## 四种满足策略

### 1. 阶梯模式
逐步满足，每次提升一层。适合升级流。

### 2. 延迟-爆发模式
长时间积累，一次性大满足。适合史诗奇幻。

### 3. 多轨并行模式
多个期待交替满足。适合多线叙事。

### 4. 颠覆-重定向模式
颠覆原期待，建立新期待。适合反套路故事。

## 输出示例

```
🎯 期待总览

【活跃期待】3 项
1. [Plot] 击败云海宗宗主 | Ch3 | 进度:45% | 🟠 High
2. [Mystery] 父亲的秘密 | Ch5 | 进度:20% | 🟢 Low
3. [Growth] 突破金丹期 | Ch15 | 进度:60% | 🔴 Critical

【建议】
💡 Ch27: 金丹小突破（部分满足）
💡 Ch28: 与宗主弟子交手（阶段性满足）
💡 Ch30: 金丹完全突破（完整 payoff）
```

## 使用方式

### 自动激活
当检测到期待相关内容时自动激活：
- 明确承诺（"我发誓……"）
- 谜题（"他是谁？"）
- 角色目标（"我要……"）

### 与其他命令配合

| 命令/Skill | 配合方式 |
|-----------|---------|
| `/plan` | 在大纲阶段识别期待，建议满足时间点 |
| `/analyze` | 提供期待维度的分析数据 |
| `/revise` | 修改时检查期待回收情况 |
| `pacing-control` | 期待满足节奏影响整体节奏 |
| `character-arc` | 角色目标 = 角色成长类期待 |

## 最佳实践

1. **明确承诺**：重要情节明确铺垫，让 Skill 能识别
2. **阶段性满足**：大期待分解为小里程碑，维持阅读动力
3. **体裁适配**：爽文频繁满足，史诗可长延迟
4. **定期检查**：每 10 章查看一次期待状态
5. **控制数量**：同时活跃的期待控制在 3-6 个
