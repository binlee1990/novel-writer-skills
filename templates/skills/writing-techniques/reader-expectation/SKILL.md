# 读者期待管理 (Reader Expectation Management)

## 快速参考（Quick Reference）

### 何时激活

**自动激活关键词**：
- 用户说"期待""承诺""悬念""铺垫""伏笔""payoff""满足""失望"
- 用户问"读者会不会等不及""这个伏笔什么时候收""会不会太慢"
- `/analyze` 发现未回收伏笔或期待管理问题
- `/plan` 阶段讨论情节节奏和高潮安排

**前置条件**：
- 至少有故事大纲或 3+ 章已完成内容
- 有明确的情节承诺或悬念设置

### 五种期待类型速查

```
情节承诺 (Plot Promise)    → "总有一天我要打败他!" → 期待决战
谜题悬念 (Mystery)         → "那个人是谁?" → 期待揭晓
角色成长 (Character Growth) → 弱者主角 → 期待变强
关系弧线 (Relationship)    → 敌对二人 → 期待和解/对决
世界观谜题 (World Mystery)  → "这个世界的真相?" → 期待揭秘
```

**核心规则**：每个承诺都必须被追踪，每个期待都必须被管理——要么满足，要么有意颠覆。

## 核心原则 (Core Principles)

### 1. 承诺必须兑现

**原则**：你对读者做出的每一个承诺，都是一份契约。

**什么算承诺？**
- 角色的誓言和目标声明
- 铺垫暗示的事件（契诃夫之枪）
- 类型惯例带来的隐含期待
- 故事开头建立的基调承诺

**违约后果**：
- 读者失望 → 弃书
- 情节漏洞 → 口碑下降
- 信任丧失 → 后续承诺无效

### 2. 延迟创造张力

**原则**：期待的价值不在于满足本身，而在于满足前的等待。

**延迟的艺术**：
- 太短 → 没有张力，满足感弱
- 太长 → 读者失去耐心，期待衰减
- 恰好 → 张力最大化，满足感最强

**延迟工具箱**：
- 增加障碍（新敌人、新困难）
- 阶段性满足（小胜利维持兴趣）
- 制造新谜题（转移注意力）
- 提供进展线索（保持希望）

### 3. 体裁决定节奏

**原则**：不同类型的读者有不同的耐心阈值。

| 体裁 | 满足频率 | 容忍延迟 | 核心期待 |
|------|---------|---------|---------|
| 爽文/升级流 | 每 3-5 章 | ≤ 10 章 | 变强、打脸、获宝 |
| 都市/现实 | 每 5-10 章 | ≤ 15 章 | 逆袭、成功、感情 |
| 悬疑/推理 | 渐进式 | ≤ 20 章 | 真相揭晓 |
| 史诗奇幻 | 每 10-15 章 | ≤ 30 章 | 大战、命运、世界 |
| 文艺/纯文学 | 不固定 | 灵活 | 角色内心、人性 |

### 4. 阶段性满足维持兴趣

**原则**：大期待分解为小里程碑，持续给读者"进展感"。

**示例：击败大反派**
```
Ch5:  打败小喽啰        → 进度 15%（能力展示）
Ch12: 击败副手          → 进度 35%（实力提升）
Ch20: 与反派正面交锋    → 进度 55%（差距缩小）
Ch28: 获得关键力量      → 进度 75%（胜利在望）
Ch35: 最终决战          → 进度 100%（完整满足）
```

每个里程碑都给读者一次小满足，维持"继续读下去"的动力。

## 期待检测 (Expectation Detection)

### 检测规则

#### 1. 情节承诺 (Plot Promise)

**关键句式**：
- "总有一天我要……"
- "我发誓……"
- "等我……之后……"
- "终有一日……"
- "我一定会……"

**隐含承诺**：
- 角色立下的目标
- 故事开头建立的冲突方向
- 反派的威胁（暗示必须被解决）

**分类**：
```
类型: plot_promise
基础紧迫性: medium
铺垫强度判断:
  - 强（大喊/发誓/重大事件）→ 紧迫性 ×3
  - 中（对话提及/目标声明）→ 紧迫性 ×2
  - 弱（心中暗想/隐约感觉）→ 紧迫性 ×1
```

#### 2. 谜题悬念 (Mystery)

**关键句式**：
- "……是谁？"
- "为什么……？"
- "……到底……"
- "真相究竟……"
- "……的秘密"

**隐含谜题**：
- 角色的神秘过去
- 未解释的异常现象
- 被刻意隐藏的信息

**分类**：
```
类型: mystery
基础紧迫性: depends on context
  - 核心谜题（主线相关）→ high
  - 次要谜题（支线/氛围）→ medium
  - 背景谜题（世界观）→ low
```

#### 3. 角色成长 (Character Growth)

**关键元素**：
- 明确的目标声明（"我的梦想是……"）
- 当前的弱点/缺陷（暗示需要克服）
- 导师的预言/期望（"你将来会……"）
- 力量体系的下一阶段（暗示突破）

**分类**：
```
类型: character_goal
基础紧迫性: low to medium（长期目标）
特殊规则: 升级流体裁中紧迫性 ×2
```

#### 4. 关系弧线 (Relationship)

**关键元素**：
- 两个角色之间的张力（敌对/暧昧/误会）
- 未说出口的感情
- 被迫分离的伙伴
- 需要和解的冲突

**分类**：
```
类型: relationship
基础紧迫性: medium
特殊规则: 言情体裁中紧迫性 ×2
```

#### 5. 世界观谜题 (World Mystery)

**关键元素**：
- "这个世界的真相是什么？"
- 未探索的区域/禁地
- 古老的传说/预言
- 力量体系的更高层次

**分类**：
```
类型: world_mystery
基础紧迫性: low
特殊规则: 史诗奇幻体裁中紧迫性 ×1.5
```

## 满足度追踪 (Fulfillment Tracking)

### 进度衡量

**情节承诺**：进度 = 目标实现的百分比
```
0%:   刚立誓/刚建立期待
25%:  初步尝试/小胜利
50%:  重大进展/中期里程碑
75%:  接近目标/最后准备
100%: 完全实现/期待满足
```

**谜题悬念**：进度 = 揭露的信息百分比
```
0%:   只知道"有秘密"
30%:  发现线索
60%:  了解部分真相
90%:  接近完整真相
100%: 完全揭晓
```

**角色成长**：进度 = 成长阶段
```
0%:   起点/瓶颈
30%:  感悟/契机
60%:  突破开始
80%:  接近完成
100%: 完全突破/蜕变
```

### 里程碑追踪

每个期待应记录关键里程碑：

```markdown
### 期待：击败云海宗宗主

| 章节 | 事件 | 进度 | 类型 |
|------|------|------|------|
| Ch3 | 立下复仇誓言 | 0% | 建立期待 |
| Ch8 | 第一次见面（被秒杀） | 10% | 强化期待 |
| Ch15 | 打败其弟子 | 30% | 阶段性满足 |
| Ch25 | 正面交手（不敌但逃脱） | 55% | 阶段性满足 |
| Ch35 | 获得上古传承 | 75% | 接近满足 |
| Ch42 | 最终决战，击败宗主 | 100% | 完整满足 |
```

### 满足模式

#### Full Payoff（完整满足）
完全满足期待，期待关闭。
- **时机**：高潮时刻、故事结尾
- **效果**：高满足感
- **示例**：最终决战击败大反派

#### Partial Fulfillment（部分满足）
给予进展但不完全满足。
- **时机**：中期里程碑
- **效果**：维持兴趣，期待继续
- **示例**：打败反派的副手

#### Subversion（颠覆期待）
有意颠覆读者期待，制造惊喜。
- **时机**：转折点
- **效果**：惊喜（处理得当）或失望（处理不当）
- **示例**：期待英雄胜利，实则惨败 → 但开启新的更强大的成长弧线
- **注意**：颠覆后必须建立新的、更有吸引力的期待

#### Re-activation（重新激活）
长期未提及的期待重新进入读者视野。
- **时机**：期待沉寂 10+ 章后
- **方法**：新线索、新威胁、角色提及
- **示例**：沉寂 15 章的"父亲的秘密"，通过发现父亲的旧信重新激活

## 紧迫性计算 (Urgency Calculation)

### 计算公式

```
紧迫性分数 = (章节间隔 / 体裁阈值) × 铺垫强度 × 类型权重
```

**参数说明**：

| 参数 | 说明 | 取值 |
|------|------|------|
| 章节间隔 | 当前章节 - 铺设章节 | 正整数 |
| 体裁阈值 | 该体裁读者的耐心上限 | 见下表 |
| 铺垫强度 | 承诺的明确程度 | 1-3 |
| 类型权重 | 期待类型的重要性 | 0.6-1.0 |

### 体裁阈值

| 体裁 | 阈值（章） | 说明 |
|------|-----------|------|
| 爽文/升级流 | 10 | 读者耐心最短 |
| 都市/现实 | 15 | 中等耐心 |
| 悬疑/推理 | 20 | 愿意等待揭晓 |
| 史诗奇幻 | 30 | 接受长期铺垫 |
| 纯文学 | 40 | 最有耐心 |

### 铺垫强度

| 强度 | 值 | 特征 |
|------|---|------|
| 弱 | 1 | 心中暗想、隐约感觉、背景描写 |
| 中 | 2 | 对话提及、目标声明、角色讨论 |
| 强 | 3 | 发誓、大喊、重大事件、核心冲突 |

### 类型权重

| 期待类型 | 权重 | 原因 |
|---------|------|------|
| plot_promise | 1.0 | 最核心，读者最在意 |
| mystery | 0.9 | 悬念驱动阅读 |
| character_goal | 0.8 | 角色认同感 |
| relationship | 0.7 | 情感投入 |
| world_mystery | 0.6 | 好奇心驱动 |

### 紧迫性等级

| 分数 | 等级 | 含义 | 建议行动 |
|------|------|------|---------|
| < 0.3 | 🟢 Low | 可继续延迟 | 偶尔提及，维持存在感 |
| 0.3-0.7 | 🟡 Medium | 需要进展 | 提供阶段性满足或新线索 |
| 0.7-1.0 | 🟠 High | 紧急 | 3-5 章内必须有实质进展 |
| > 1.0 | 🔴 Critical | 必须满足 | 立即安排满足或重新激活 |

### 计算示例

**示例 1：爽文复仇承诺**
```
章节间隔: 15 章
体裁阈值: 10 章（爽文）
铺垫强度: 3（大喊发誓）
类型权重: 1.0（plot_promise）

紧迫性 = (15/10) × 3 × 1.0 = 4.5 → 🔴 Critical!
建议: 必须立即满足或提供重大进展
```

**示例 2：史诗奇幻世界观谜题**
```
章节间隔: 25 章
体裁阈值: 30 章（史诗奇幻）
铺垫强度: 1（隐约暗示）
类型权重: 0.6（world_mystery）

紧迫性 = (25/30) × 1 × 0.6 = 0.5 → 🟡 Medium
建议: 提供新线索或部分揭露
```

## 节奏策略 (Pacing Strategies)

### Strategy 1: 阶梯模式 (Ladder Pattern)

逐步满足，每次提升一层。

**适用**：升级流、成长型故事

```
Ch5:  打败 E 级敌人    ████░░░░░░  10%
Ch10: 打败 D 级敌人    ██████░░░░  25%
Ch15: 打败 C 级敌人    ████████░░  45%
Ch25: 打败 B 级敌人    █████████░  70%
Ch40: 打败 A 级 Boss   ██████████  100%
```

**优点**：持续满足感，读者始终有进展感
**风险**：可能单调，需要变化（偶尔失败/意外）
**适配建议**：每 3-5 个阶梯插入一次意外（失败、新发现）

### Strategy 2: 延迟-爆发模式 (Delay-Payoff)

长时间积累张力，一次性大满足。

**适用**：史诗奇幻、严肃文学

```
Ch1:  建立期待         █░░░░░░░░░  5%
Ch2-25: 准备、历练     ██░░░░░░░░  20%（缓慢积累）
Ch26-28: 最终准备      ████░░░░░░  40%（加速）
Ch29: 决战前夜         ███████░░░  70%（张力顶点）
Ch30: 最终决战         ██████████  100%（爆发满足）
```

**优点**：满足感极强，高潮有力
**风险**：中期可能失去读者
**适配建议**：中期必须有阶段性小满足维持兴趣

### Strategy 3: 多轨并行模式 (Multi-Track)

多个期待交替满足，节奏丰富。

**适用**：多线叙事、群像剧

```
期待A（复仇）:  Ch1 铺设 ──────── Ch15 满足
期待B（爱情）:       Ch5 铺设 ──────── Ch20 满足
期待C（真相）:            Ch10 铺设 ──────── Ch25 满足
期待D（成长）:  Ch1 铺设 ────────────────── Ch30 满足
```

**优点**：节奏丰富，总有期待在被满足
**风险**：管理复杂，容易遗忘
**适配建议**：同时活跃的期待不超过 5 个

### Strategy 4: 颠覆-重定向模式 (Subversion-Redirect)

颠覆原期待，建立更有吸引力的新期待。

**适用**：转折型故事、反套路

```
Ch1-15:  期待"击败恶龙"        ████████░░  80%
Ch16:    发现恶龙是盟友！       ██░░░░░░░░  颠覆！
Ch17:    新期待"击败真正的敌人"  ██░░░░░░░░  20%（新起点）
Ch18-30: 追踪新期待             ██████████  100%
```

**优点**：惊喜感强，故事深度增加
**风险**：处理不当会让读者愤怒
**适配建议**：
- 颠覆前必须有足够的伏笔暗示
- 新期待必须比旧期待更有吸引力
- 不要频繁颠覆（每部作品最多 1-2 次重大颠覆）

## 实践应用 (Best Practices)

### 期待分析输出格式

```markdown
## 🎯 期待总览

### 活跃期待 [X] 项

| # | 类型 | 内容 | 铺设 | 进度 | 紧迫性 | 状态 |
|---|------|------|------|------|--------|------|
| 1 | Plot | 击败云海宗宗主 | Ch3 | 45% | 🟠 High (0.8) | 需要进展 |
| 2 | Mystery | 父亲的秘密 | Ch5 | 20% | 🟢 Low (0.2) | 正常延迟 |
| 3 | Growth | 突破金丹期 | Ch15 | 60% | 🔴 Critical (1.2) | 必须满足 |
| 4 | Relationship | 与师姐的关系 | Ch8 | 30% | 🟡 Medium (0.5) | 需要进展 |

### 满足度分析

🔴 **紧急**：
- "突破金丹期" — 已延迟 12 章，铺垫强度高，建议 3 章内满足

🟠 **注意**：
- "击败宗主" — 需要提供阶段性进展（建议：与宗主弟子交手）

🟢 **良好**：
- "父亲的秘密" — 渐进披露中，节奏合理
- "与师姐的关系" — 张力积累中，暂不需要满足

### 下一步建议

| 章节 | 建议 | 对应期待 | 预期效果 |
|------|------|---------|---------|
| Ch27 | 金丹小突破（部分满足） | 突破金丹期 | 进度 → 80% |
| Ch28 | 与宗主弟子交手（阶段性满足） | 击败宗主 | 进度 → 55% |
| Ch30 | 金丹完全突破（完整 payoff） | 突破金丹期 | 进度 → 100% ✅ |
| Ch32 | 发现父亲的旧信（新线索） | 父亲的秘密 | 进度 → 35% |

### 已满足期待 [X] 项

| # | 内容 | 满足章节 | 满足方式 |
|---|------|---------|---------|
| 1 | 打败外门弟子 | Ch10 | Full Payoff |
| 2 | 获得灵剑 | Ch18 | Full Payoff |
```

### 期待健康度检查

定期（每 10 章）运行健康度检查：

```markdown
## 期待健康度报告

### 总体评分：7.5/10

| 指标 | 状态 | 说明 |
|------|------|------|
| 活跃期待数 | ✅ 4 个 | 合理范围（3-6） |
| Critical 期待 | ⚠️ 1 个 | 需要尽快处理 |
| 遗忘期待 | ✅ 0 个 | 无长期未提及的期待 |
| 满足频率 | ✅ 每 8 章 | 符合体裁要求 |
| 期待多样性 | ✅ 4 种类型 | 类型丰富 |

### 风险提示
⚠️ "突破金丹期"紧迫性已达 Critical，如果 3 章内不满足，
   读者可能感到拖沓。建议在 Ch27-30 安排突破情节。

### 建议
💡 当前期待管理整体良好，注意处理 Critical 项即可。
💡 Ch30 后可以建立 1-2 个新期待，维持阅读动力。
```

## 常见问题

### Q: 期待太多怎么办？

**A**: 同时活跃的期待建议控制在 3-6 个：
- < 3 个：故事缺乏驱动力，建议建立新期待
- 3-6 个：理想范围
- > 6 个：管理困难，建议合并或满足部分期待

### Q: 读者期待和作者意图不一致怎么办？

**A**: 这是"期待错位"问题：
- **预防**：在故事开头明确基调（类型承诺）
- **处理**：如果要颠覆，必须有足够伏笔
- **示例**：如果你写的是悲剧结局，前期不要给太多"大团圆"暗示

### Q: 如何处理"契诃夫之枪"？

**A**: 契诃夫之枪 = 隐含的情节承诺：
- 如果你描写了墙上的枪，读者期待它会被使用
- 要么使用它（满足期待）
- 要么有意不使用（颠覆期待，但需要理由）
- 不要无意中遗忘（这是 bug，不是 feature）

### Q: 颠覆期待会不会让读者生气？

**A**: 取决于执行质量：
- ✅ **好的颠覆**：有伏笔铺垫 + 新期待更有吸引力 + 逻辑自洽
- ❌ **坏的颠覆**：毫无铺垫 + 为颠覆而颠覆 + 逻辑不通
- 经验法则：如果读者回头看能发现线索，就是好颠覆

### Q: 不同 POV 的期待怎么管理？

**A**: 每个 POV 角色可以有独立的期待线：
- 主角的期待 = 主线期待（最高优先级）
- 配角的期待 = 支线期待（较低优先级）
- 多 POV 时，交替满足不同角色的期待

### 爽点密度统计

**爽点定义**：让读者产生满足感/快感的情节事件

**爽点类型分类**：
| 类型 | 描述 | 典型场景 | 爽感强度 |
|------|------|---------|---------|
| 打脸 | 主角碾压看不起他的人 | 宗门大比、退婚后重逢 | ⭐⭐⭐⭐⭐ |
| 升级 | 境界/能力突破 | 闭关突破、战斗中领悟 | ⭐⭐⭐⭐ |
| 获得 | 得到珍贵资源/法宝 | 秘境探险、拍卖会 | ⭐⭐⭐ |
| 逆袭 | 从劣势翻盘 | 绝境反击、废物觉醒 | ⭐⭐⭐⭐⭐ |
| 揭秘 | 重要真相揭示 | 身世之谜、阴谋揭露 | ⭐⭐⭐⭐ |
| 认可 | 获得他人认可/尊重 | 师父认可、敌人敬佩 | ⭐⭐⭐ |

**统计方法**：
1. 逐章扫描内容，识别爽点事件
2. 标记爽点类型和强度
3. 计算密度和间隔
4. 与类型基准对比（网文建议 ≥ 0.8/章，文学作品 ≥ 0.3/章）

### 信息密度分析

**信息类型**：
- **世界观信息**：新的设定、规则、历史
- **角色信息**：新角色登场、角色背景揭示、角色关系变化
- **情节信息**：新线索、新冲突、新目标

**密度计算**：
```
信息密度 = 新信息点数 / 章节字数 × 1000
```

**健康范围**：
- 网文：2-5 信息点/千字（太少无聊，太多信息过载）
- 文学：1-3 信息点/千字

### 与期待管理的整合

当进行读者体验分析时，将爽点和信息投喂与期待管理结合：
- 爽点是否对应了某个期待的满足？
- 信息释放是否推进了某个悬念？
- 是否有期待被满足但没有产生爽感？（满足方式不够精彩）
