# 钩子设计 (Hook Design)

## 快速参考

### 核心定义

**钩子**：章节结尾处让读者产生「必须继续读下去」冲动的叙事技巧。

**核心公式**：钩子 = 未完成的期待 + 即将发生的变化

**黄金法则**：每章结尾必须有钩子。没有钩子的章节 = 读者的退出点。

### 六种钩子类型速查

```
悬念钩子 (Suspense)     → "门外传来脚步声。" → 读者想知道是谁
反转钩子 (Reversal)     → "他笑了。因为那封信是假的。" → 颠覆认知
信息钩子 (Information)  → "她终于想起来了——那个人，她见过。" → 关键信息即将揭示
情感钩子 (Emotional)    → "他转身离开。她没有追。" → 情感悬而未决
时间压力钩子 (Urgency)  → "还有三天。三天后，一切都来不及了。" → 倒计时紧迫感
选择钩子 (Dilemma)      → "救左边，还是救右边？" → 两难抉择
```

## 核心原则

### 1. 钩子的本质是「未完成」

**原则**：人类大脑天然厌恶未完成的事物（蔡格尼克效应）。钩子利用这一心理，在章节结尾制造「未完成感」。

**三种未完成**：
- **动作未完成**：正在发生的事被打断（打斗中断、对话中断）
- **信息未完成**：关键信息即将揭示但未揭示（"他终于知道了真相——"）
- **情感未完成**：情感冲突悬而未决（告白未回应、背叛未对质）

### 2. 钩子强度分级

| 强度 | 描述 | 适用场景 | 示例 |
|------|------|---------|------|
| ⭐ | 轻度好奇 | 日常过渡章 | "明天会是怎样的一天呢。" |
| ⭐⭐ | 中度期待 | 发展章节 | "他决定去找那个人谈谈。" |
| ⭐⭐⭐ | 强烈悬念 | 冲突升级章 | "门被踹开了。" |
| ⭐⭐⭐⭐ | 震撼反转 | 高潮前章 | "那个人摘下面具。是他。" |
| ⭐⭐⭐⭐⭐ | 命运转折 | 卷末/关键节点 | "他死了。至少所有人都这么认为。" |

**节奏规律**：
- 不能每章都用 ⭐⭐⭐⭐⭐，读者会麻木
- 建议分布：60% 中度（⭐⭐-⭐⭐⭐）、30% 强烈（⭐⭐⭐-⭐⭐⭐⭐）、10% 震撼（⭐⭐⭐⭐⭐）
- 连续 3 章以上低强度钩子 = 读者流失风险

### 3. 钩子必须兑现

**原则**：钩子是对读者的承诺。下一章必须回应钩子，否则读者会感到被欺骗。

**兑现方式**：
- **直接兑现**：下一章开头直接揭示（最常见）
- **延迟兑现**：切换视角，制造更大悬念后再揭示（高级技巧，慎用）
- **部分兑现**：揭示部分信息，制造新的悬念（连环钩子）

**禁忌**：
- ❌ 钩子制造的悬念在下一章被轻描淡写地化解
- ❌ 钩子暗示的危险完全没有发生
- ❌ 连续多章不回应之前的钩子

## 六种钩子类型详解

### 1. 悬念钩子 (Suspense Hook)

**机制**：在章节结尾制造一个未知的威胁或事件，读者必须翻页才能知道结果。

**适用场景**：动作场景、危险逼近、未知来访

**写法要点**：
- 用短句制造紧张感
- 聚焦感官细节（声音、气味、触感）
- 不要解释，只呈现

**示例**：

❌ **弱悬念**：
```
他感觉到有人在跟踪他，心里有些不安，但还是决定继续走下去。
```

✅ **强悬念**：
```
身后的脚步声停了。

他没有回头。

但他知道，那个人还在。
```

### 2. 反转钩子 (Reversal Hook)

**机制**：在章节最后一刻颠覆读者（和角色）的认知，制造「原来如此」的冲击。

**适用场景**：阴谋揭露、身份暴露、计划失败/成功

**写法要点**：
- 前文必须有足够的铺垫（否则反转会显得突兀）
- 反转信息用最短的句子呈现
- 让读者和角色同时震惊

**示例**：

❌ **弱反转**：
```
他终于打开了那封信，发现信的内容和他想的完全不一样，
原来这一切都是一个精心策划的骗局。
```

✅ **强反转**：
```
他打开信。

三个字。

"我知道。"
```

### 3. 信息钩子 (Information Hook)

**机制**：暗示关键信息即将揭示，但在揭示前切断章节。

**适用场景**：谜题解答、秘密发现、记忆恢复

**写法要点**：
- 角色已经「几乎」知道了答案
- 用省略或中断制造悬念
- 信息的重要性必须在前文有铺垫

**示例**：
```
她盯着那张照片看了很久。

突然，她想起来了。

那个人——她在哪里见过？

对了。是在——
```

### 4. 情感钩子 (Emotional Hook)

**机制**：在情感高潮或转折点切断章节，让读者的情感悬而未决。

**适用场景**：告白、分离、背叛、和解

**写法要点**：
- 聚焦角色的情感反应（而非事件本身）
- 用沉默和留白制造张力
- 不要在章节内解决情感冲突

**示例**：
```
"我喜欢你。"

她说完这句话，转身就走。

他站在原地。雨越下越大。

他没有追。
```

### 5. 时间压力钩子 (Urgency Hook)

**机制**：在章节结尾引入或强化时间限制，制造紧迫感。

**适用场景**：倒计时、截止日期、即将到来的威胁

**写法要点**：
- 明确的时间节点
- 后果必须严重
- 当前进度明显不足

**示例**：
```
还有七十二小时。

七十二小时后，封印就会彻底崩溃。

而他们，连第一道门都还没打开。
```

### 6. 选择钩子 (Dilemma Hook)

**机制**：在章节结尾让角色面临两难选择，读者想知道角色会怎么选。

**适用场景**：道德困境、利益冲突、忠诚考验

**写法要点**：
- 两个选项都有代价
- 没有明显的「正确答案」
- 选择的后果必须重大

**示例**：
```
左边是师父。右边是她。

毒雾还有三分钟就会蔓延到这里。

他只能救一个。
```

## 钩子链设计

### 什么是钩子链

**定义**：多个章节的钩子形成连锁反应，前一个钩子的兑现触发下一个钩子，形成持续的阅读驱动力。

### 钩子链模式

#### 模式 1：递进式（最常用）

```
第 N 章钩子：发现线索 →
第 N+1 章钩子：线索指向某人 →
第 N+2 章钩子：找到此人，但此人已死 →
第 N+3 章钩子：死因指向更大的阴谋
```

**特点**：每个钩子比前一个更强，悬念持续升级

#### 模式 2：分支式

```
第 N 章钩子 A：主角发现秘密
第 N 章钩子 B：反派开始行动（切换视角）
→ 两条线各自发展，读者知道它们终将交汇
```

**特点**：利用信息差制造紧张感

#### 模式 3：回旋式

```
第 N 章钩子：危机出现
第 N+1 章：危机解决，但引发新问题
第 N+2 章钩子：新问题比原来更严重
```

**特点**：「解决一个问题，制造两个问题」，适合升级流

### 钩子链规划表

| 章节 | 钩子类型 | 强度 | 内容 | 兑现章节 | 触发的下一个钩子 |
|------|---------|------|------|---------|---------------|
| 第 1 章 | 悬念 | ⭐⭐⭐ | [描述] | 第 2 章 | 信息钩子 |
| 第 2 章 | 信息 | ⭐⭐ | [描述] | 第 3 章 | 反转钩子 |
| 第 3 章 | 反转 | ⭐⭐⭐⭐ | [描述] | 第 5 章 | 时间压力钩子 |

## 网文钩子特殊技巧

### 1. 爽点钩子

**网文特有**：在爽点高潮处切断，让读者为了看「打脸结果」而继续。

```
他一掌拍出。

所有人都看到了——那道金光。

宗主的脸色变了。
```

### 2. 升级钩子

**网文特有**：在突破/升级的关键时刻切断。

```
丹田中的灵力开始沸腾。

第九层。

他感觉到了。就差一点——
```

### 3. 打脸钩子

**网文特有**：在反派嘲讽后、主角即将反击时切断。

```
"就凭你？一个废物？"

他笑了。

慢慢抬起手。
```

## 常见陷阱

### 1. 虚假钩子

❌ 制造悬念但下一章完全不相关
- 读者会感到被欺骗，信任度下降

### 2. 钩子疲劳

❌ 每章都用最强钩子
- 读者会麻木，钩子失效

### 3. 钩子与内容脱节

❌ 钩子暗示的方向与实际发展不符
- 比如钩子暗示大战，下一章却是日常

### 4. 过度依赖悬念钩子

❌ 只会用「门外有脚步声」类型
- 应该混合使用六种钩子类型

### 5. 钩子过于明显

❌ "接下来会发生什么呢？让我们拭目以待。"
- 这不是钩子，这是旁白

## 检测清单

### 每章写完后自检

- [ ] 本章结尾是否有明确的钩子？
- [ ] 钩子类型是什么？（悬念/反转/信息/情感/时间压力/选择）
- [ ] 钩子强度是否与章节定位匹配？
- [ ] 上一章的钩子是否在本章得到回应？
- [ ] 最近 3 章的钩子类型是否有变化？（避免单一）
- [ ] 钩子是否与主线情节相关？（避免无意义悬念）

### 每卷规划时检查

- [ ] 是否有钩子链设计？
- [ ] 卷末钩子是否足够强（⭐⭐⭐⭐ 以上）？
- [ ] 钩子强度分布是否合理（不是一直高强度）？
- [ ] 是否有连续 3 章以上的低强度钩子？（风险点）
