# Archetype Application Guide (原型应用指南)

## 使用流程

### Step 1：识别故事需求

在选择原型之前，先问自己：

**关于故事：**
- 这个故事的核心冲突是什么？（决定了 Hero 和 Shadow 的关系）
- 故事的基调是什么？（严肃？幽默？混合？——决定了是否需要 Trickster）
- 故事的主题是什么？（成长？救赎？爱？——决定了原型的选择和组合）

**关于角色：**
- 主角需要什么样的成长？（决定了 Hero 的类型）
- 主角需要指导吗？（决定了是否需要 Mentor）
- 需要制造不确定性吗？（决定了是否需要 Shapeshifter）
- 需要喜剧缓解吗？（决定了是否需要 Trickster）

### Step 2：选择原型

根据需求匹配原型：

| 故事需求 | 推荐原型 | 说明 |
|---------|---------|------|
| 主角成长 | Hero | 必备——没有英雄就没有故事 |
| 指导和传承 | Mentor | 加速英雄成长，传递智慧 |
| 核心冲突 | Shadow | 制造考验，推动英雄改变 |
| 情感支持 | Ally | 陪伴英雄，提供互补能力 |
| 不确定性 | Shapeshifter | 制造悬念，挑战信任 |
| 喜剧元素 | Trickster | 缓解紧张，揭露真相 |
| 阶段考验 | Threshold Guardian | 验证英雄资格，控制节奏 |
| 启动旅程 | Herald | 打破现状，传递召唤 |
| 权威冲突 | Father | 规则与自由的张力 |
| 情感深度 | Mother | 无条件的爱与保护 |
| 天真视角 | Child | 希望、新生、纯真 |
| 哲学深度 | Sage | 智慧、洞察、超然 |

### Step 3：建立基础框架

为每个角色确定：

1. **主原型** — 他在故事中的核心功能是什么？
2. **核心特征** — 参考对应原型文档的"核心特征"部分
3. **典型弧线** — 参考对应原型文档的"典型弧线"部分
4. **与其他角色的关系** — 对立？互补？转变？

### Step 4：添加独特性

原型是骨架，独特性是血肉。为每个角色添加：

- **特殊背景：** 独特的文化、职业、经历
- **独特缺陷：** 非典型的弱点或矛盾
- **个性化细节：** 怪癖、习惯、说话方式
- **非典型弧线：** 颠覆传统成长路径
- **内在矛盾：** 让角色的不同面向互相冲突

**关键原则：** 独特性应该与原型产生张力，而不是完全脱离原型。一个有恐高症的英雄比一个完全不像英雄的"英雄"更有趣。

### Step 5：检查团队平衡

- 是否覆盖了故事需要的所有功能？
- 角色之间是否有足够的差异？
- 是否有有趣的对立和互补关系？
- 是否有角色功能重复？（如果有，考虑合并或删除）

## 不同体裁的原型应用

### 爽文/升级流

**核心原型配置：**
- **必备：** Hero（成长明显、升级清晰）、Mentor（传授技能、提供金手指）
- **常用：** Threshold Guardian（关卡 boss、升级考验）、Ally（队友、后宫成员）
- **少用：** Trickster（可能干扰爽感）、Shapeshifter（可能制造不必要的复杂性）

**特点：**
- Hero 的成长是线性的、可量化的（等级、技能、装备）
- Shadow 通常是阶梯式的——一个比一个强的对手
- Mentor 可能有多个，每个阶段一个

**注意：**
- 避免 Hero 过于被动（只靠金手指，不做选择）
- Shadow 不能太弱（否则没有爽感）也不能太强（否则需要外挂解决）

### 史诗奇幻

**核心原型配置：**
- **必备：** Hero、Mentor、Shadow、Ally 团队
- **常用：** Herald、Threshold Guardian、Sage
- **特点：** 原型齐全，关系复杂，多线叙事

**特点：**
- 通常有完整的英雄之旅结构
- 配角团队覆盖多种原型
- Shadow 往往是宏大的——不只是个人对手，而是世界级威胁

**经典配置：** 《魔戒》模式——Hero + Mentor + Ally 团队 + 终极 Shadow

### 悬疑推理

**核心原型配置：**
- **必备：** Hero（侦探）、Shadow（凶手/幕后黑手）
- **常用：** Shapeshifter（嫌疑人）、Herald（报案人/线索提供者）
- **特点：** 大量 Shapeshifter 制造不确定性

**特点：**
- 几乎所有配角都有 Shapeshifter 的特质——每个人都可能是凶手
- Shadow 通常隐藏在 Shapeshifter 的面具下
- Hero（侦探）的核心能力是看穿 Shapeshifter 的伪装

**注意：**
- Mentor 在推理中通常弱化——侦探靠自己的能力
- Trickster 可以是有趣的配角（提供错误线索或喜剧缓解）

### 浪漫爱情

**核心原型配置：**
- **必备：** Hero（主角）、Shapeshifter（爱情对象——忠诚和感情不确定）
- **常用：** Threshold Guardian（反对恋情的力量——父母、社会、前任）、Ally（闺蜜/兄弟）
- **特点：** Shapeshifter 是核心原型

**特点：**
- 爱情对象几乎总是 Shapeshifter——"他/她到底爱不爱我？"就是 Shapeshifter 的核心张力
- Threshold Guardian 制造外部障碍——不是所有阻碍都来自内心
- Shadow 可能是情敌、前任或主角自己的恐惧

**注意：**
- 避免让爱情对象只是 Shapeshifter——他们也需要自己的原型深度
- Ally（闺蜜/兄弟）是重要的情感出口——主角需要有人倾诉

### 武侠/仙侠

**核心原型配置：**
- **必备：** Hero（主角）、Mentor（师父）、Shadow（魔头/对手）
- **常用：** Father（掌门/宗主）、Threshold Guardian（关卡守护者）、Ally（师兄弟）
- **特点：** 师承关系是核心，Father 原型特别重要

**特点：**
- Mentor 和 Father 常常合一（师父既是导师也是权威）
- 修炼体系本身就是一系列 Threshold Guardian
- Shadow 常与 Hero 有师出同门的关系（增加镜像性）

### 现实主义/文学小说

**核心原型配置：**
- **必备：** Hero（但可能是反英雄或被动主角）
- **常用：** Shadow（但可能是内在的或体制性的）、Shapeshifter
- **特点：** 原型更隐蔽，角色更复杂

**特点：**
- 原型不那么明显——角色更接近真实的人
- Shadow 常常是内在的（主角自己的缺陷）或体制性的（社会压力）
- 没有明确的 Mentor——主角必须自己摸索
- 成长可能是微妙的、模糊的，甚至是负面的

## 常见陷阱

### 陷阱 1：机械套用

❌ **错误：** 生硬套用原型模板，角色像从流水线上下来的
✅ **正确：** 原型是起点和工具，不是终点。用原型建立框架，然后用独特性让角色活起来

**检测方法：** 如果你能用原型名称完全替代角色名称（"导师说了一句话"而不是"甘道夫说了一句话"），那角色还不够独特。

### 陷阱 2：原型过载

❌ **错误：** 试图让一个角色体现所有原型——他既是导师又是英雄又是骗子又是阴影
✅ **正确：** 一个主原型 + 最多两个次原型。少即是多

**检测方法：** 如果你无法用一句话描述角色的核心功能，他可能承载了太多原型。

### 陷阱 3：忽视原型功能

❌ **错误：** 完全不考虑角色在故事中的功能，导致角色"好看但没用"
✅ **正确：** 每个角色都应该有明确的叙事功能——如果去掉这个角色，故事应该有明显的缺失

**检测方法：** 问自己"如果删掉这个角色，故事会怎样？"如果答案是"没什么变化"，这个角色需要重新设计。

### 陷阱 4：原型冲突

❌ **错误：** 让角色同时对同一个英雄扮演矛盾的原型（既是导师又是阴影），但没有叙事理由
✅ **正确：** 如果角色确实有矛盾的原型面向，这应该是有意的设计，并且在故事中被探索和解决

**例外：** 邓布利多对哈利既是 Mentor 又有 Shadow 的特质（操纵、隐瞒）——但这是有意的设计，是整个系列的核心张力之一。

### 陷阱 5：原型刻板化

❌ **错误：** Mentor 一定是白胡子老人，Shadow 一定穿黑衣服，Trickster 一定是小个子
✅ **正确：** 原型是功能，不是外表。任何外表的角色都可以扮演任何原型

**检测方法：** 如果你的角色外表和原型完全匹配（导师=老人，英雄=年轻人），考虑颠覆一下。

## 快速参考卡

### 我需要什么原型？

| 如果你的故事需要... | 考虑使用... |
|-------------------|-----------|
| 一个成长的主角 | Hero |
| 有人教主角 | Mentor |
| 一个强大的对手 | Shadow |
| 可靠的伙伴 | Ally |
| 不确定谁可以信任 | Shapeshifter |
| 笑声和轻松时刻 | Trickster |
| 阶段性的考验 | Threshold Guardian |
| 启动故事的事件 | Herald |
| 权威和规则的压力 | Father |
| 无条件的爱和保护 | Mother |
| 天真和希望 | Child |
| 深层的智慧和洞察 | Sage |

### 最小可行团队

- **最简配置（3人）：** Hero + Shadow + Ally
- **标准配置（5人）：** Hero + Mentor + Shadow + Ally + Shapeshifter
- **完整配置（8人）：** 以上 + Trickster + Threshold Guardian + Herald
