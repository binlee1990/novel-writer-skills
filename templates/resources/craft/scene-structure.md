# 场景结构写作知识库

## 快速参考（Quick Reference）

场景是小说的基本构建单元——不是章节，而是"有明确目标和结果的完整行动序列"。

**Scene-Sequel 模式**：
```
Scene (行动) → Sequel (反应) → Scene (新行动) → ...
```

**Scene 结构**（主动、向前推进）：
1. **Goal（目标）** - 角色想要什么
2. **Conflict（冲突）** - 什么阻碍了角色
3. **Disaster（灾难）** - 失败或意外转折

**Sequel 结构**（被动、消化后果）：
1. **Reaction（反应）** - 情感反应
2. **Dilemma（困境）** - 分析选项
3. **Decision（决定）** - 做出选择

**核心公式**：
```
优秀场景 = 明确目标 + 升级冲突 + 意外灾难 → 反应 → 新决定 → 新场景
```

**不是场景**：
- ❌ 只是描写环境或心理活动（缺乏行动和冲突）
- ❌ 目标模糊或没有明显结果
- ❌ 角色轻易达成目标（缺乏张力）

## 核心原则（Core Principles）

### 1. Scene（场景）- 行动单元

Scene 是故事的引擎，是角色追求目标、遭遇障碍的地方。

#### A. Goal（目标）

**原则**：每个场景开始时，角色必须有明确的目标。

**好目标的特征**：
- **具体可见** - 读者能看到角色在追求什么
- **有时限** - 即时的、当下的目标（不是"成为有钱人"）
- **与主情节相关** - 推进故事主线或角色成长

❌ **糟糕的目标**（模糊）：
```
林晓想要改善自己的生活。
```

✅ **优秀的目标**（具体）：
```
林晓必须在下午三点前说服老板给她加薪。
```

**目标类型**：

| 类型 | 描述 | 示例 |
|------|------|------|
| 获取信息 | 得到关键情报 | "我要问出真相" |
| 完成行动 | 达成具体任务 | "我要逃出这栋楼" |
| 改变他人 | 说服、威胁、欺骗 | "我要让他签合同" |
| 保护 | 阻止某事发生 | "我不能让她发现" |
| 到达地点 | 物理移动 | "我必须赶到机场" |

**检查清单**：
- [ ] 读者能在场景开头识别出角色的目标吗？
- [ ] 目标是否具体且可衡量？
- [ ] 目标是否有足够的重要性（Stakes）？

#### B. Conflict（冲突）

**原则**：目标和结果之间必须有阻碍。没有冲突 = 没有故事。

**冲突的三种类型**：

**1. 人物冲突**（最常见，最有力）

```
【目标】林晓要说服老板加薪
【冲突】老板拒绝："公司今年预算紧张。"
【升级】林晓提出有竞争对手挖她
【再冲突】老板："那你去吧。"
```

**2. 环境冲突**

```
【目标】主角要逃出废墟
【冲突】门被堵死
【升级】余震开始
【再冲突】氧气耗尽
```

**3. 内心冲突**

```
【目标】主角要杀死仇人
【冲突】仇人已经失去记忆，变成了好人
【困境】复仇还是放手？
```

**冲突升级的节奏**：

```
小障碍 → 中等障碍 → 严重障碍 → 几乎不可能
```

❌ **糟糕的冲突**（轻易解决）：
```
"门锁了。"
她从包里拿出钥匙，打开了门。
```

✅ **有力的冲突**（真正的障碍）：
```
"门锁了。"
她试了所有钥匙，都不对。
砸门？会惊动守卫。
撬锁？她不会。
窗户？太高了。
脚步声传来，越来越近……
```

**冲突的强度匹配**：
- **低张力场景**：小冲突即可（日常对话的误会）
- **中张力场景**：中等冲突（重要谈判受阻）
- **高潮场景**：极端冲突（生死抉择）

#### C. Disaster（灾难）

**原则**：场景不能以"角色轻易达成目标"结束。必须有转折。

**灾难的三种形式**：

**1. "No"（完全失败）**

```
【目标】说服老板加薪
【结果】老板拒绝，还降职了她
```

**2. "Yes, but"（达成目标，但代价高昂）**

```
【目标】逃出大楼
【结果】逃出了，但队友被抓了
```

**3. "No, and"（失败，而且情况更糟）**

```
【目标】阻止炸弹爆炸
【结果】没阻止，而且发现还有第二颗炸弹
```

**灾难的作用**：
- 制造悬念："然后怎么办？"
- 改变角色处境（更糟或更复杂）
- 引出 Sequel（反应和新决定）

❌ **糟糕的场景结尾**（没有转折）：
```
他去见客户，成功签下了合同。一切顺利。
```

✅ **有力的场景结尾**（灾难转折）：
```
他去见客户，成功签下了合同。
离开时，他的手机震动：
"你签的是假客户，这是个陷阱。"
```

**灾难的强度**：

| 强度 | 描述 | 何时使用 |
|------|------|---------|
| 小挫折 | 延迟但未摧毁希望 | 开篇、日常场景 |
| 重大失败 | 计划完全失败 | 中期转折点 |
| 毁灭性打击 | 失去重要的人/物 | 高潮前的至暗时刻 |

---

### 2. Sequel（续场）- 反应单元

Sequel 是 Scene 和下一个 Scene 之间的桥梁，是角色消化后果和做出新决定的地方。

#### A. Reaction（情感反应）

**原则**：灾难发生后，角色的即时情感反应。

**反应的层次**：

**身体反应**（即时）：
```
她的手开始颤抖。
```

**情感反应**（几秒后）：
```
愤怒、恐惧、绝望涌上心头。
```

**理性反应**（稍后）：
```
"这不可能……"她试图理解发生了什么。
```

❌ **过度理性**（缺乏情感）：
```
计划失败了。她立刻开始分析下一步该怎么办。
```

✅ **真实反应**（情感先行）：
```
计划失败了。
她盯着空荡荡的房间，脑子一片空白。
怎么会这样？
她靠在墙上，慢慢滑坐到地上。
许久，她才重新站起来。
```

**反应的时长**：
- **轻微灾难** → 简短反应（几句话）
- **重大灾难** → 详细反应（几段）
- **毁灭性打击** → 深度反应（可能是一整章）

#### B. Dilemma（困境）

**原则**：角色分析选项，权衡利弊，但没有完美选择。

**真正的困境**：
- 所有选项都有代价
- 没有明显的"正确答案"
- 角色必须做出艰难选择

❌ **虚假困境**（有明显最优解）：
```
选项 A：冒险救人，可能成功
选项 B：什么都不做，肯定后悔
→ 答案太明显了
```

✅ **真正困境**（两难）：
```
选项 A：去救队友，可能导致任务失败，更多人死亡
选项 B：完成任务，队友必死
→ 无论选哪个都有巨大代价
```

**困境的呈现方式**：

**内心独白**：
```
去还是不去？
去，任务就完了。
不去，她就死了。
她是我的搭档，是我把她带进来的……
但如果任务失败，几百人会死……
```

**与他人讨论**：
```
"我们得回去救她。"
"回去？你疯了？"
"我们不能丢下她！"
"那任务呢？那些无辜的人呢？"
```

**困境的时长**：
- 快节奏故事 → 简短困境（几段）
- 慢节奏故事 → 详细困境（深入分析）

#### C. Decision（决定）

**原则**：做出选择，设定新目标，进入下一个 Scene。

**好决定的特征**：
- **明确** - 不是"我再想想"，而是"我决定要……"
- **行动导向** - 引出下一个具体目标
- **有后果** - 这个决定会影响后续情节

❌ **无力的决定**：
```
她决定想办法解决这个问题。
```

✅ **有力的决定**：
```
她决定直接去找老板摊牌。
明天一早，无论结果如何。
```

**决定→新目标→新Scene**：

```
【Disaster】计划A失败，队友被抓
【Reaction】震惊、自责
【Dilemma】救人 vs 完成任务
【Decision】决定救人
→ 【新目标】潜入敌营救出队友（新Scene开始）
```

### 3. Scene-Sequel 的节奏组合

#### 标准节奏

```
Scene (行动) → Sequel (反应) → Scene (新行动) → Sequel (反应) → ...
```

**示例**：
```
Scene 1: 主角去面试（目标），被刁难（冲突），没拿到offer（灾难）
Sequel 1: 沮丧（反应），纠结要不要继续找工作（困境），决定转行（决定）
Scene 2: 主角去学新技能（新目标），遇到严格导师（冲突），第一天被赶出来（灾难）
Sequel 2: 愤怒（反应），考虑放弃 vs 坚持（困境），决定证明自己（决定）
```

#### 快节奏：省略 Sequel

在高度紧张的场景中，可以省略 Sequel，让 Scene 连续发生。

```
Scene 1: 逃离追捕
Scene 2: 躲进小巷
Scene 3: 被堵住
Scene 4: 跳窗逃走
→ 连续行动，没时间反应
```

**何时省略 Sequel**：
- 动作序列（追逐、打斗）
- 极度紧急的情况
- 多米诺骨牌式的连锁事件

**警告**：不要过度省略，否则读者没有情感喘息的空间。

#### 慢节奏：拉长 Sequel

在深度情感场景中，Sequel 可以比 Scene 更长。

```
Scene: 主角收到父亲去世的消息（灾难）
Sequel: 详细的悲伤反应、回忆、困境、决定
→ Sequel 可以是一整章
```

**何时拉长 Sequel**：
- 重大情感打击
- 角色成长的关键时刻
- 故事主题的深度探讨

### 4. 场景的开始和结束

#### 场景开始的技巧

**原则**：尽快让读者了解目标和情境。

✅ **立即进入行动**：
```
林晓推开会议室的门。
"我要加薪。"
```

❌ **过度铺垫**：
```
林晓今天心情不错。她早上起床，吃了早餐，想了很久，决定去找老板。
她穿过走廊，注意到窗外的天气……
（200字后）
终于，她推开会议室的门。
```

**开场的要素**：
- **时间/地点** - 快速定位
- **角色目标** - 让读者知道这场景要干什么
- **初始情境** - 设定基调

**开场模板**：

**1. 行动开场**
```
她踹开了门。
```

**2. 对话开场**
```
"你不能进去！"
"看我的。"
```

**3. 内心驱动开场**
```
她必须拿到那份文件，无论付出什么代价。
```

#### 场景结束的技巧

**原则**：在高点结束，制造悬念。

✅ **悬念结尾**：
```
她打开信封。
看到里面的内容，她的脸色瞬间煞白。
```

✅ **决定性行动结尾**：
```
"好。"她深吸一口气，"那我们就这么干。"
```

✅ **时间切断结尾**：
```
电话响了。
她接起来，听到的第一句话是：
"他们知道了。"
```

❌ **过度总结结尾**：
```
这次谈话让她明白了很多道理。她决定改变自己的生活方式。
第二天，她开始实施计划……
（继续详细描述）
```

**结尾的目标**：
- 让读者想翻到下一页
- 提出新的问题
- 改变情境或角色状态

## 实践应用（Practical Application）

### 在 /specify 阶段

**定义场景类型和风格**

```markdown
## 场景设计原则

- 主要场景类型：冲突对话场景、行动场景
- Scene-Sequel 比例：70% Scene / 30% Sequel
- 平均场景长度：1500-2500 字
- 灾难强度分布：
  - 开篇：小挫折
  - 中期：重大失败
  - 高潮：毁灭性打击
```

### 在 /plan 阶段

**规划每个场景的结构**

```markdown
第 5 章场景设计

Scene 1: 谈判
- Goal: 说服投资人
- Conflict: 投资人怀疑数据
- Disaster: 投资人发现主角隐瞒了关键信息

Sequel 1:
- Reaction: 恐慌、自责
- Dilemma: 坦白 vs 继续隐瞒
- Decision: 决定先发制人

Scene 2: 主动出击
- Goal: 抢先公布真相
- Conflict: 媒体不相信
- Disaster: 竞争对手捷足先登
```

### 在 /write 阶段

**写作时的检查清单**：

每个 Scene 写完后：
- [ ] 目标是否明确？读者能看出角色想要什么吗？
- [ ] 冲突是否升级？还是轻易解决了？
- [ ] 结尾有灾难转折吗？还是太顺利了？
- [ ] 场景是否推进了主情节？

每个 Sequel 写完后：
- [ ] 反应是否真实？是否给了角色情感反应的时间？
- [ ] 困境是否艰难？是否真的没有完美选择？
- [ ] 决定是否引出下一个明确目标？

### 在 /analyze 阶段

**分析场景质量**：

```markdown
## 场景分析

### 结构检查
- Scene-Sequel 模式完整性：✅
- 目标明确性：8/10 场景明确，2/10 需要强化
- 灾难转折：所有 Scene 都有转折 ✅

### 问题场景
第 3 章 Scene 2: 目标模糊
- 问题：角色"想找线索"太笼统
- 建议：改为"必须在警察到达前找到证据"
```

## 常见陷阱（Common Pitfalls）

### 1. 目标模糊或缺失

**问题**：场景中角色没有明确想要达成的东西。

❌ **糟糕示例**：
```
林晓在咖啡馆坐了一个下午，看着窗外的人来人往，
思考着人生的意义。
```

**问题分析**：
- 没有具体目标
- 没有冲突
- 没有结果
- 这不是场景，只是描写

✅ **改进示例**：
```
林晓必须在张伟到达前想出一个合理的解释。
她坐在咖啡馆，在纸上写下又划掉各种说辞。
"家里有事"？太假了。
"突然生病"？查得出来。
"——他来了。
她还没想好。
```

**改进**：
- 明确目标：想出解释
- 有时限：张伟到达前
- 有冲突：想不出来
- 有灾难：时间到了还没想好

### 2. 冲突太弱

**问题**：障碍轻易被克服，没有真正的对抗。

❌ **糟糕示例**：
```
她需要进入那栋楼。
门锁着。
她找到一扇窗户，翻了进去。
```

✅ **改进示例**：
```
她需要进入那栋楼。
门锁着，窗户都被封死了。
她试图撬锁——警报响起。
她跑到后门——也锁着。
守卫的脚步声越来越近……
最后，她不得不躲进垃圾箱。
```

**改进**：
- 多重障碍
- 升级的冲突
- 代价（藏在垃圾箱）

### 3. 场景以成功结束

**问题**：角色轻易达成目标，没有转折。

❌ **糟糕示例**：
```
他去找投资人，展示了商业计划。
投资人很满意，当场签约。
一切都很顺利。
```

**问题**：
- 没有悬念
- 读者失去期待
- 故事没有推进（太顺了）

✅ **改进示例**（三种方式）：

**No**：
```
投资人听完后摇头："太冒险了。"
合同没签成。
```

**Yes, but**：
```
投资人签了约："但你必须在一个月内完成产品。"
这几乎不可能。
```

**No, and**：
```
投资人拒绝了："而且，你的竞争对手刚拿到了我们的全部投资。"
```

### 4. 缺少 Sequel

**问题**：一个 Scene 直接跳到下一个 Scene，没有反应和决定。

❌ **糟糕示例**：
```
Scene 1: 主角的父亲去世了（重大灾难）
Scene 2: 主角去参加商业会议
→ 缺少情感处理
```

✅ **改进示例**：
```
Scene 1: 主角的父亲去世了
Sequel: 主角的悲伤、回忆、面对是否继续经营父亲生意的困境
Scene 2: 主角决定继续，去参加父亲原本要参加的会议
```

**何时必须有 Sequel**：
- 重大情感事件后
- 关键决策点
- 角色成长时刻

### 5. 过长的 Sequel

**问题**：过度的内心独白和分析，停滞不前。

❌ **糟糕示例**：
```
她坐在那里想了很久。
她想起小时候的事，想起母亲说过的话，
想起那个夏天，想起初恋，想起……
（3000 字的回忆和思考）
```

**问题**：
- 行动停滞
- 读者失去耐心
- 偏离主情节

✅ **改进示例**：
```
她坐在那里，脑海中闪过母亲的话：
"永远不要放弃你爱的人。"
她站起来。
她知道该怎么做了。
```

**原则**：
- Sequel 应该简洁
- 只包含必要的反应和决定
- 快速回到行动（新 Scene）

### 6. 所有场景结构相同

**问题**：每个场景都是标准的 Goal-Conflict-Disaster，变得机械。

**解决方法**：

**变化 1：省略 Sequel**（快节奏）
```
Scene → Scene → Scene → Sequel
```

**变化 2：拉长 Sequel**（深度情感）
```
Scene → 长 Sequel → Scene
```

**变化 3：多目标场景**
```
Goal A → Conflict A → 达成 Goal A
→ Goal B 出现 → Conflict B → Disaster B
```

**变化 4：反转**
```
看似会成功 → 突然失败
看似会失败 → 意外成功（但有代价）
```

## 示例分析（Examples）

### 案例 1：标准 Scene 结构

**场景**：求职面试

```
【Goal】
林晓走进面试室。
她必须拿到这份工作，房租还有三天就到期了。

【Conflict - 初期】
"你的简历上有两年的空白期。"面试官皱眉。
"我在照顾生病的母亲。"
"那期间你的专业技能……"
"我有自学，这是我的作品集。"

【Conflict - 升级】
面试官翻了翻作品集，表情不置可否。
"你的期望薪资是多少？"
"八千。"
"我们只能给六千。"
"但市场价……"
"这是我们的预算。"

【Disaster】
"我们会考虑的。"面试官站起来，"两周内给你答复。"
两周？她等不了两周。
离开时，她看到外面还有十几个面试者在等待。
```

**分析**：
- **明确目标**：拿到工作
- **Stakes（利害关系）**：房租快到期
- **冲突升级**：简历质疑 → 薪资谈判
- **灾难类型**："Yes, but"（可能拿到，但要等两周且薪资低）
- **引出悬念**：她等不了两周，下一步怎么办？

### 案例 2：Sequel 的处理

**承接上个场景的 Disaster**

```
【Reaction - 身体/情感】
林晓走出大楼，靠在墙上。
手在颤抖。
两周。
她只有三天。

【Dilemma】
她可以：
1. 借钱付房租——但已经欠了太多人情
2. 搬到更便宜的地方——但没时间找房
3. 接受之前那份不正规的工作——违法边缘
4. 回老家——承认失败

【Decision】
她拿出手机，拨通了那个号码。
"喂？我是林晓。那份工作……我接受。"
即使知道这可能是个错误。
```

**分析**：
- **真实反应**：先是身体反应（颤抖），后是理性思考
- **真正困境**：所有选项都有严重代价
- **明确决定**：接受有问题的工作
- **引出新 Scene**：新工作的场景

### 案例 3：快节奏省略 Sequel

**场景**：逃亡序列

```
【Scene 1】
Goal: 逃出大楼
Conflict: 守卫封锁了主要出口
Disaster: 被发现，警报响起

【Scene 2 - 无 Sequel，直接下一个目标】
Goal: 躲避追捕
Conflict: 走廊太长，无处可躲
Disaster: 被堵在死胡同

【Scene 3 - 仍然无 Sequel】
Goal: 找到逃生路线
Conflict: 窗户锁死了
Disaster: 守卫已经到达

【Scene 4】
Goal: 最后一搏
Conflict: 寡不敌众
Disaster: 被抓住

【Sequel - 终于出现】
她被押进房间。
心跳如雷。
完了。
门打开，走进来的人让她呆住——
是她的搭档。
```

**分析**：
- **连续 Scene**：模拟紧张的逃亡感
- **没有时间反应**：每个 Scene 立即引出下一个
- **最后的 Sequel**：给读者喘息，并制造新的转折

## 进阶技巧（Advanced Techniques）

### 1. 多角色场景的目标管理

**挑战**：一个场景中有多个角色，每个都有不同目标。

**解决方法**：确定主要视点角色的目标为场景主目标

**示例**：
```
【场景】：商业谈判
【主视点】：林晓
【林晓的目标】：签下合同
【对方的目标】：压低价格
【第三方的目标】：从中获利

→ 场景以林晓的目标为主线，其他目标制造冲突
```

### 2. Scene 内的微转折

不是只在场景结尾有 Disaster，Scene 内部也可以有小转折。

**结构**：
```
Goal → Conflict → 小成功 → 新障碍 → 再冲突 → Disaster
```

**示例**：
```
Goal: 说服老板加薪
Conflict: 老板拒绝
小成功: 老板说"让我考虑考虑"
新障碍: 竞争对手插话"我也想要加薪"
再冲突: 老板说"那你们竞争吧，谁业绩好加给谁"
Disaster: 被迫进入竞争，而对方实力更强
```

**效果**：
- 增加波折
- 延长张力
- 更真实（现实中目标不会直线失败）

### 3. 反转灾难的性质

**传统灾难**：坏事发生

**反转**：好事发生，但读者知道这是坏事

**示例**：
```
Goal: 赢得比赛
Conflict: 对手很强
Result: 赢了！（看似成功）
Disaster: 但读者知道，赢了意味着她会被选中去执行危险任务
```

**戏剧反讽**：角色高兴，读者担忧。

### 4. 延迟 Disaster 的揭露

**技巧**：Scene 似乎成功结束，下一个 Scene 才揭露灾难。

**示例**：
```
Scene 1 结尾:
"合同签了，我们成功了！"大家欢呼。

Scene 2 开头（几小时后）:
"老板，出事了。"
"什么事？"
"合同里有一条隐藏条款……"
```

**效果**：
- 制造虚假安全感
- 让 Disaster 更震撼

### 5. 嵌套场景结构

**概念**：大 Scene 包含多个小 Scene

**示例：宴会场景**
```
大目标: 在宴会上找到盟友

小Scene 1:
- Goal: 接近目标人物
- Conflict: 被竞争对手挡住
- Disaster: 目标人物走了

小Scene 2:
- Goal: 在舞池中再次接近
- Conflict: 目标人物的保镖阻拦
- Disaster: 被怀疑是刺客

大Disaster:
被赶出宴会，失去所有机会
```

## 检查清单（Checklist）

### Scene 检查

#### 目标清晰度
- [ ] 场景开头，角色的目标是否明确？
- [ ] 读者能轻易识别角色想要什么吗？
- [ ] 目标是否具体、可见、有时限？
- [ ] 目标是否与主情节相关？

#### 冲突强度
- [ ] 是否有真正的障碍阻止角色达成目标？
- [ ] 冲突是否升级（而非轻易解决）？
- [ ] 冲突强度是否匹配场景重要性？
- [ ] 是否有至少 2-3 层障碍？

#### 灾难转折
- [ ] 场景是否以灾难/转折结束？
- [ ] 角色是否轻易达成目标（应该避免）？
- [ ] 灾难类型（No / Yes,but / No,and）是否合理？
- [ ] 灾难是否制造悬念，引出下一个场景？

### Sequel 检查

#### 反应真实性
- [ ] 角色是否有情感反应（而非立即理性思考）？
- [ ] 反应时长是否匹配灾难严重程度？
- [ ] 是否展示了身体→情感→理性的反应层次？

#### 困境质量
- [ ] 是否存在真正的困境（多个坏选项）？
- [ ] 是否有一个明显最优解（应该避免）？
- [ ] 困境是否推进角色成长或主题？

#### 决定明确性
- [ ] 决定是否明确、具体？
- [ ] 决定是否引出下一个 Scene 的明确目标？
- [ ] 决定是否有后果？

### 整体节奏
- [ ] Scene 和 Sequel 的比例是否合适？
- [ ] 是否在需要时省略了 Sequel（快节奏）？
- [ ] 是否在需要时拉长了 Sequel（深度情感）？
- [ ] 场景变化是否有节奏感（而非机械重复）？

## 与七步方法论的集成

### /constitution - 宪法阶段

设定场景设计的基本原则：

```markdown
## 场景设计原则
- 采用 Scene-Sequel 模式
- 每个 Scene 必须有明确的 Goal-Conflict-Disaster
- Scene/Sequel 比例：70/30（行动优先）
- 避免轻易成功，保持张力
```

### /specify - 详细设定

定义具体的场景规范：

```markdown
## 场景规范

### 平均场景长度
- 行动场景：2000-3000 字
- 对话场景：1500-2000 字
- 内心场景：1000-1500 字

### 灾难强度分布
- 开篇（1-5章）：小挫折为主
- 中期（6-15章）：中等失败，偶尔重大打击
- 高潮前（16-18章）：毁灭性打击
- 高潮（19-20章）：生死抉择
```

### /plan - 规划阶段

为每个章节规划 Scene 结构：

```markdown
第 3 章：突破困境

Scene 1: 林晓主动出击
- Goal: 找到投资人的弱点
- Conflict: 投资人滴水不漏
- Disaster: 反被投资人拿到了林晓的黑料

Sequel 1:
- Reaction: 恐慌
- Dilemma: 认输 vs 破釜沉舟
- Decision: 决定揭露投资人的秘密

Scene 2: 媒体发布会
- Goal: 公开投资人的秘密
- Conflict: 媒体不相信
- Disaster: 投资人反过来起诉林晓
```

### /tasks - 任务分解

将 Scene 作为任务单元：

```markdown
- [ ] 写 Scene 1: 主动出击（Goal-Conflict-Disaster）
- [ ] 写 Sequel 1: 恐慌与决定
- [ ] 写 Scene 2: 媒体发布会
```

### /write - 写作阶段

写作时实时检查 Scene 结构：
- 每写完一个 Scene，检查 Goal-Conflict-Disaster 是否完整
- 确保灾难转折存在
- 验证目标的明确性

### /analyze - 分析阶段

分析场景结构质量：

```markdown
## 场景结构分析

### 统计
- 总场景数：15
- Scene/Sequel 比例：11 Scene / 4 Sequel = 73/27 ✅
- 平均 Scene 长度：2100 字 ✅

### 问题场景
第 5 章 Scene 2:
- 问题：目标不明确（"想要变强"太模糊）
- 建议：改为具体目标（"必须通过晋级考核"）

第 8 章：缺少 Sequel
- 问题：重大灾难后直接跳到下一 Scene
- 建议：添加反应和决定的 Sequel
```

## 场景类型快速参考

### 行动场景

**结构**：快节奏，省略 Sequel
```
Goal: 逃出 → Conflict: 追击 → Disaster: 被困
```

**技巧**：
- 短句
- 快速来回
- 连续 Scene

### 对话场景

**结构**：标准 Scene-Sequel
```
Goal: 说服 → Conflict: 争执 → Disaster: 破裂
Reaction: 后悔 → Dilemma: 道歉 vs 坚持 → Decision: 决定
```

**技巧**：
- 对话即冲突
- 潜台词
- 情感升级

### 内心场景

**结构**：拉长 Sequel
```
Disaster（前一场景的）
→ 长 Reaction（深度反思）
→ Dilemma（道德困境）
→ Decision（重要选择）
```

**技巧**：
- 内心独白
- 回忆
- 自我质疑

### 调查场景

**结构**：微转折密集
```
Goal: 找线索
→ 找到线索A → 引出新谜题
→ 找到线索B → 发现之前理解错了
→ Disaster: 真相比想象的更糟
```

**技巧**：
- 逐步揭露
- 线索误导
- 真相反转

## 总结（Summary）

**场景构建的黄金法则**：

1. **每个 Scene 有明确的 Goal-Conflict-Disaster**
2. **不要让角色轻易成功** - 制造障碍和转折
3. **Scene-Sequel 交替** - 行动与反应的节奏
4. **灾难引出悬念** - "然后怎么办？"
5. **困境必须艰难** - 没有完美选择

**记住**：
- 场景是故事的引擎，不是描写的容器
- 每个场景都应该改变情境
- Disaster 不是惩罚角色，而是推进故事
- Sequel 不是浪费篇幅，而是情感深度
- 结构是框架，不是束缚——可以变化和打破

**最终测试**：
删除一个场景，故事是否仍然成立？
如果是，这个场景可能不够重要。
如果不是，这个场景是必要的。

**经典公式**：
```
Goal (目标) + Conflict (冲突) + Disaster (灾难) = Scene
Reaction (反应) + Dilemma (困境) + Decision (决定) = Sequel
Scene + Sequel = 完整的故事单元
```

掌握场景结构 = 掌握故事的基本构建方法。
