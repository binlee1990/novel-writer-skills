# 张力管理 (Tension Management)

## 快速参考

### 核心定义

**张力**：读者在阅读过程中感受到的紧张感、期待感和不确定感。张力是驱动读者继续阅读的核心动力。

**张力公式**：张力 = 风险 × 不确定性 × 读者关心程度

### 四种张力来源

```
1. 信息不对称 — 读者知道角色不知道的（或反过来）
2. 时间压力   — 倒计时、截止日期、即将到来的威胁
3. 两难选择   — 没有完美答案的困境
4. 期待落差   — 读者期待 A，实际发生 B
```

---

## 信息不对称

### 类型

| 类型 | 描述 | 张力来源 | 示例 |
|------|------|---------|------|
| 读者知道，角色不知道 | 戏剧性反讽 | 读者替角色着急 | 读者知道杯中有毒，角色即将喝下 |
| 角色知道，读者不知道 | 悬念 | 读者想知道真相 | 角色看到信后表情大变，但不说内容 |
| 角色 A 知道，角色 B 不知道 | 人际张力 | 读者期待揭露时刻 | A 知道 B 的秘密身份 |
| 所有人都不知道 | 谜题 | 读者想解谜 | 连环失踪案，无人知道凶手 |

### 使用技巧

- **逐步释放**：不要一次性揭示所有信息，分 3-5 次释放
- **误导性线索**：释放部分真实信息 + 部分误导信息
- **揭示时机**：在读者最想知道的时候延迟揭示，在读者快要失去耐心时揭示
- **多层信息差**：同时维持 2-3 个不同的信息不对称

### 常见错误

- ❌ 信息差维持太久，读者失去兴趣
- ❌ 揭示太突然，没有铺垫
- ❌ 揭示后张力完全消失（应该用新的信息差替代）

## 时间压力

### 设计原则

1. **明确的截止时间**：读者必须知道「还剩多少时间」
2. **严重的后果**：超时的后果必须让读者在意
3. **可见的进度**：读者能感受到时间在流逝
4. **意外的阻碍**：计划总是不如变化快

### 时间压力模板

| 类型 | 时间尺度 | 适用场景 |
|------|---------|---------|
| 即时压力 | 秒-分钟 | 战斗、逃跑、拆弹 |
| 短期压力 | 小时-天 | 任务期限、比赛倒计时 |
| 中期压力 | 周-月 | 修炼期限、敌人来袭 |
| 长期压力 | 年 | 诅咒倒计时、世界末日 |

### 递进技巧

```
初始：还有 30 天
中期：还有 7 天（发现新障碍）
后期：还有 3 天（计划失败，需要新方案）
高潮：还有 3 小时（最后一搏）
```

## 两难选择

### 设计原则

1. **两个选项都有代价**：没有明显的「正确答案」
2. **代价类型不同**：不是「大代价 vs 小代价」，而是「A 类代价 vs B 类代价」
3. **选择揭示角色**：角色的选择体现其价值观
4. **选择有后果**：选择之后，未选的那条路的代价也要体现

### 两难类型

| 类型 | 描述 | 示例 |
|------|------|------|
| 利益冲突 | 两个都想要，只能选一个 | 救师父还是救爱人 |
| 价值冲突 | 两种价值观的碰撞 | 遵守承诺还是做正确的事 |
| 时间冲突 | 两件事都紧急，只能做一件 | 阻止炸弹还是追凶手 |
| 信息不足 | 不知道哪个选择更好 | 两扇门，一扇通向安全，一扇通向死亡 |

### 写法要点

- 给读者足够的时间感受两难（不要一笔带过）
- 展示角色的内心挣扎
- 选择后不要立即化解代价（让代价持续影响后续剧情）

## 期待与意外平衡

### 核心原则

**满足期待让读者满意，打破期待让读者惊喜。最好的故事两者兼有。**

### 期待管理矩阵

| | 读者期待发生 | 读者期待不发生 |
|---|------------|--------------|
| **实际发生** | 满足感（爽点） | 意外/反转 |
| **实际不发生** | 悬念延续 | 确认（安心） |

### 技巧

1. **70/30 法则**：70% 满足期待，30% 打破期待
2. **超越期待**：不只是满足，而是比期待的更好（打脸比预想的更狠）
3. **延迟满足**：让读者等待，但最终给出更大的满足
4. **假反转**：看似打破期待，实际是更大满足的铺垫

## 张力节奏

### 张力曲线模板

```
章节内张力曲线（单章）：
开头：中等张力（承接上章钩子）
发展：张力逐步上升
高点：张力峰值（冲突/揭示/决定）
结尾：张力不完全释放（钩子）

多章张力曲线（一卷）：
第 1-3 章：建立基础张力
第 4-10 章：张力波动上升（小高潮 + 短暂释放）
第 11-15 章：张力持续攀升
第 16-20 章：张力峰值（大高潮）
第 21-25 章：张力释放 + 新张力种子
```

### 张力叠加

同时维持多条张力线，当一条释放时另一条接管：

```
张力线 A（主线冲突）：  ████████████████░░░░░░░░
张力线 B（角色关系）：  ░░░░████████████████░░░░
张力线 C（谜题悬念）：  ░░░░░░░░████████████████
                       ↑ 始终有至少一条张力线活跃
```

## 检测清单

### 每章检查
- [ ] 本章是否有至少一种张力来源？
- [ ] 张力是否在章节结尾保持（不完全释放）？
- [ ] 是否有信息不对称在推进？

### 每卷检查
- [ ] 张力是否整体呈上升趋势？
- [ ] 是否有多条张力线交替活跃？
- [ ] 两难选择是否有后续影响？
- [ ] 时间压力是否被合理使用？
